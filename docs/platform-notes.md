# Platform Notes

## Recommended setup

- Public website: GitHub Pages
- Waitlist: Wix Forms, Brevo, MailerLite, Google Forms, or Formspree
- Future community: self-hosted Discourse on a VPS
- Future subdomain: community.throupletea.com
- Email: existing Wix / OutAt Inc email setup is fine for public-facing contact

## Why not Facebook

The goal is to own the audience relationship. Facebook groups are easy, but the platform owns discovery, delivery, rules, and access. This project should collect emails and member context from day one.

## Why not custom software yet

The first product is not software. The first product is culture. The site and waitlist validate demand before spending money on infrastructure.

## Cheapest professional path

1. Launch this static GitHub Pages site.
2. Point a clean domain or subdomain to it.
3. Replace the form action with your waitlist tool.
4. Collect 100 qualified invite requests.
5. Launch a private founding beta.
6. Only then pay for a VPS/community backend.

## Future options

### Discourse
Best long-term serious community option. Strong moderation, trust levels, private messages, categories, and plugins.

### NodeBB
More modern/social feel, good real-time features, but smaller ecosystem.

### Flarum
Beautiful and lightweight, but less mature for a larger serious community.
